System Overview

The project models a university information system with students, employees, courses, grading, news notifications, employee requests, authentication, and research activity. Most domain behavior is placed in model classes, while service classes coordinate authentication and research queries. Persistent in-memory collections are centralized in a singleton database repository.

User management Course registration Teacher grading Reports Research papers Serialization
  • Model layer: contains business entities such as User, Student, Teacher, Course, and ResearchPaper.
  • Service layer: provides application operations through AuthenticationService, ResearchService, comparators, and factories.
  • Repository layer: stores users, courses, projects, news, and requests through the singleton Database.

Top 10 Classes

User

project/model/User.java
Abstract model

Base abstraction for every person who can use the university system.

Detailed Description

User centralizes identity, contact data, school membership, login/logout behavior, and notification handling. It implements Observer, so all users can receive updates when a News item is published.

Attributes / Properties

  • String id: unique user identifier.
  • String passwordHash: stored password hash used by authentication.
  • String firstName, String lastName, String email: personal profile fields.
  • School school: user's school, defaulting to School.GENERAL.
  • List<String> notifications: private notification messages received from news updates.

Initialization / Constructor

Protected constructor: User(String id, String passwordHash, String firstName, String lastName, String email). It is called by subclasses such as Student and Employee.

Methods / Brief Overview

login() and logout() print session messages, update(News) stores news notifications, and getFullName() combines first and last name.

Usage Example

Student student = new Student("S001", "hash123", "Aruzhan", "Kim",
        "aruzhan@uni.kz", "Computer Science", 2, 0, 3.6);
student.setSchool(School.SITE);
System.out.println(student.getFullName());

Employee

project/model/Employee.java
Abstract model

Shared base class for all university staff members.

Detailed Description

Employee extends User with employment data and staff-only workflows. It supports direct messaging between employees and creates employee requests that are automatically registered in the singleton Database.

Attributes / Properties

  • double salary: employee salary value.
  • LocalDate hireDate: date when the employee was hired.
  • List<Message> messages: private inbox and sent-message history.

Initialization / Constructor

Protected constructor: Employee(String id, String passwordHash, String firstName, String lastName, String email, double salary, LocalDate hireDate). Concrete subclasses provide public constructors.

Methods / Brief Overview

sendMessage(...) sends and stores staff messages, createRequest(String) creates a formal request, and getters/setters expose salary and hire date.

Usage Example

Manager manager = new Manager("M001", "hash", "Dana", "Omar",
        "dana@uni.kz", 550000, LocalDate.now(), ManagerType.OR);
EmployeeRequest request = manager.createRequest("Need approval for event budget.");

Student

project/model/Student.java
Model

Represents a student and controls course registration, marks, GPA, and research status.

Detailed Description

Student contains most of the academic learner workflow. It validates registration against course major/year rules and a 21-credit limit, tracks pending and approved courses, recalculates GPA from transcript marks, and can become a researcher through a delegated DefaultResearcher role.

Attributes / Properties

  • String major, int year, DegreeLevel degreeLevel: academic profile.
  • int credits, double gpa: academic progress metrics.
  • Transcript transcript: course-to-mark record storage.
  • Researcher researcherRole, Researcher supervisor: optional research participation fields.
  • List<Course> registeredCourses, pendingCourses: enrollment state.

Initialization / Constructor

Student(String id, String passwordHash, String firstName, String lastName, String email, String major, int year, int credits, double gpa). The degree level defaults to BACHELOR, and a new Transcript is created automatically.

Methods / Brief Overview

Registration is handled by registerForCourse and approveCourseRegistration. Academic records use viewTranscript, viewMarks, and updateMark. Research status uses becomeResearcher and assignSupervisor.

Usage Example

Student student = new Student("S002", "hash", "Ali", "Nurseit",
        "ali@uni.kz", "Software Engineering", 1, 0, 0.0);
Course oop = new Course("CS101", "Object-Oriented Programming", 5,
        "Software Engineering", 1);
student.registerForCourse(oop);

Teacher

project/model/Teacher.java
Model

Represents teaching staff who manage courses, grading, ratings, and complaints.

Detailed Description

Teacher extends employee behavior with teaching responsibilities. It verifies that a teacher is assigned to a course before viewing students or putting marks. Professors automatically receive a researcher role.

Attributes / Properties

  • TeacherTitle title: teacher rank such as tutor, lecturer, or professor.
  • List<Course> courses: courses assigned to the teacher.
  • Researcher researcherRole: optional research behavior.
  • Map<Student, Integer> ratings: student ratings used for average rating.

Initialization / Constructor

Teacher(String id, String passwordHash, String firstName, String lastName, String email, double salary, LocalDate hireDate, TeacherTitle title). If title is PROFESSOR, the teacher becomes a researcher automatically.

Methods / Brief Overview

putMark calculates and writes grades, viewStudents returns enrolled students, sendComplaint sends a message to a manager, and getAverageRating summarizes ratings.

Usage Example

Teacher teacher = new Teacher("T001", "hash", "Mira", "Saken",
        "mira@uni.kz", 480000, LocalDate.now(), TeacherTitle.LECTOR);
teacher.addCourse(oop);
teacher.putMark(student, oop, new Mark(27, 25, 36));

Manager

project/model/Manager.java
Model

Coordinates academic administration tasks for courses, reports, requests, and news.

Detailed Description

Manager is the main administrative role for academic operations. It uses the singleton Database to publish courses, gather report data, retrieve signed employee requests, and send news updates to all users.

Attributes / Properties

  • ManagerType type: manager category, such as OR or department manager.
  • Inherited employee fields: salary, hire date, and messages.

Initialization / Constructor

Manager(String id, String passwordHash, String firstName, String lastName, String email, double salary, LocalDate hireDate, ManagerType type).

Methods / Brief Overview

approveRegistration moves pending courses into a student's registered list, assignCourse links teacher and course, createReport summarizes system statistics, and manageNews publishes announcements.

Usage Example

Manager manager = new Manager("M002", "hash", "Aida", "Serik",
        "aida@uni.kz", 600000, LocalDate.now(), ManagerType.DEPARTMENT);
manager.addCourseForRegistration(oop, "Software Engineering", 1);
manager.approveRegistration(student);

Admin

project/model/Admin.java
Model

Maintains user accounts and records administrative actions.

Detailed Description

Admin is responsible for account-level operations. It delegates storage changes to Database and keeps a local log showing which users were added, removed, or updated.

Attributes / Properties

  • List<String> logs: private action history for admin operations.
  • Inherited employee fields: identity, salary, hire date, and messages.

Initialization / Constructor

Admin(String id, String passwordHash, String firstName, String lastName, String email, double salary, LocalDate hireDate).

Methods / Brief Overview

addUser, removeUser, and updateUser modify database records. viewLogs returns a defensive copy of the admin log.

Usage Example

Admin admin = new Admin("A001", "hash", "Nurlan", "Bek",
        "admin@uni.kz", 650000, LocalDate.now());
admin.addUser(student);
System.out.println(admin.viewLogs());

Course

project/model/Course.java
Model

Defines a university course and the people and lessons attached to it.

Detailed Description

Course stores academic course metadata and relationships. Its availability rule checks optional intended major and intended year values before allowing a student registration request.

Attributes / Properties

  • String code, String name: course identity and title.
  • int credits: credit value used in student credit-limit checks.
  • String intendedMajor, Integer intendedYear: optional registration restrictions.
  • List<Teacher> instructors, List<Student> students, List<Lesson> lessons: course relationships.

Initialization / Constructor

Use Course(String code, String name, int credits) for a general course, or Course(String code, String name, int credits, String intendedMajor, Integer intendedYear) to restrict registration.

Methods / Brief Overview

addInstructor, addStudent, and addLesson attach related objects. isAvailableFor(Student) validates registration eligibility.

Usage Example

Course algorithms = new Course("CS205", "Algorithms", 5,
        "Computer Science", 2);
algorithms.addInstructor(teacher);
boolean allowed = algorithms.isAvailableFor(student);

Database

project/repository/Database.java
Repository

Singleton repository that stores and serializes all main application data.

Detailed Description

Database is a final singleton repository backed by Java serialization. It loads from database.ser at startup when possible, returns defensive copies for collection getters, and synchronizes public operations to reduce concurrent access issues.

Attributes / Properties

  • static Database INSTANCE: singleton instance loaded from disk or newly created.
  • Map<String, User> users: users indexed by ID.
  • Map<String, Course> courses: courses indexed by code.
  • List<ResearchProject> researchProjects, List<News> newsFeed, List<EmployeeRequest> employeeRequests: stored domain collections.

Initialization / Constructor

The constructor is private. Use Database.getInstance(). On first load, the class tries to deserialize database.ser; otherwise it creates an empty database.

Methods / Brief Overview

Provides synchronized CRUD-style operations for users and courses, plus add/get methods for research projects, news, employee requests, and save() for persistence.

Usage Example

Database db = Database.getInstance();
db.addUser(student);
db.addCourse(oop);
db.save();

AuthenticationService

project/service/AuthenticationService.java
Service

Validates user credentials against the database.

Detailed Description

AuthenticationService is a small service object that separates login checks from the model layer. It compares a provided password hash with the stored hash, calls login() on success, and supports both optional and exception-based authentication flows.

Attributes / Properties

  • Database database: repository used to find users by ID.

Initialization / Constructor

AuthenticationService(Database database). The caller supplies the database instance, usually Database.getInstance().

Methods / Brief Overview

authenticate(String, String) returns Optional<User>. requireAuthenticated(String, String) returns a user or throws SecurityException.

Usage Example

AuthenticationService auth = new AuthenticationService(Database.getInstance());
User currentUser = auth.requireAuthenticated("S002", "hash");

ResearchService

project/service/ResearchService.java
Service

Coordinates research papers, researchers, citation searches, and research projects.

Detailed Description

ResearchService works across a collection of users and extracts researcher behavior from direct Researcher implementations or delegated roles on students and teachers. It can locate top-cited researchers globally, by school, by year, or by school and year.

Attributes / Properties

  • No instance fields. The service is stateless and operates on method parameters.
  • Uses Researcher, ResearchPaper, ResearchProject, User, and School objects supplied by callers.

Initialization / Constructor

Uses the default no-argument constructor: new ResearchService().

Methods / Brief Overview

addPaper appends a publication, getAllResearchers extracts research roles, findTopCitedResearcher... methods rank researchers by citations, and addResearcherToProject validates project membership.

Usage Example

ResearchService researchService = new ResearchService();
student.becomeResearcher();
researchService.addPaper(student.getResearcherRole(), paper);
researchService.findTopCitedResearcher(Database.getInstance().getUsers())
        .ifPresent(r -> System.out.println(r.getResearcherName()));

Important Relationships

User hierarchy

User is the root. Student extends it directly, while Admin, Manager, and Teacher inherit through Employee.

Academic workflow

Student requests a Course, Manager approves registration, and Teacher records Mark values into the student's transcript.

Research workflow

ResearchService works with the Researcher interface and can discover researcher roles from students, teachers, and research employees.
